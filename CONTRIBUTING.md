# Contributing

## Workflow
- Branch from `main`.
- Use short-lived branches named `feat/...`, `fix/...`, `chore/...`, `docs/...`, `refactor/...`, or `test/...`.
- Open one pull request per coherent change.
- Use a conventional commit-style PR title such as `feat: add meeting scheduling`.
- Squash merge into `main` after required checks pass.
- Delete the merged branch.

## Local Checks
- `pnpm lint`
- `pnpm typecheck`
- `pnpm build`
